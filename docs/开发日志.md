**实现内容:**

1. **财务比率计算 (33个)** — 从三张财报（balance_sheet + income_statement + cash_flow）按 code + REPORT_DATE 内连接后向量化计算：
   - 流动性: current_ratio, quick_ratio, cash_ratio
   - 杠杆: debt_to_assets, debt_to_equity, equity_multiplier, long_term_debt_ratio, equity_to_assets
   - 偿付: interest_coverage
   - 盈利: roe, roa, gross_margin, operating_margin, net_margin, ebitda_margin

   - 现金流: cf_to_revenue, cf_to_debt, cf_to_assets, cf_to_netprofit, free_cf
   - 资产结构: fixed_asset/intangible/goodwill/inventory/receivables_ratio
   - 营运: working_capital_to_assets
   - 每股: eps, bps

2. **同比增长率 (8个)** — revenue/netprofit/assets/equity/operate_profit/operating_cf/total_liabilities_yoy，按 code 分组 shift(4) 计算

3. **周转率 (4个)** — inventory/receivable/asset/equity_turnover，使用当期与上期均值分母

4. **ST 违约标签** — 尝试 akshare `stock_st_pool` (不存在此API)，回退使用 `TRACE_上市公司基本信息.csv` 的 `is_st` 字段。正样本 255 只 ST 股票，占比 4.83%

5. **缺失值处理** — 行业中位数 → 全局中位数 → 0 三级填充；宏观列前向填充

**关键 Bug 修复:**

## 2026-05-27: 数据质量修复 (P0/P1/P2)

**变更文件:**
- `src/features.py` — P0 (ST时间序列标签，多数据源) + P2 (Winsorize)
- `src/market_quarterly.py` — P1 (后缀归一化 guard)
- `data/namechange_history.csv` — P0 新增缓存 (akshare SZSE ST记录)
- `data/market_quarterly.csv` — P1 重新生成 (正確后缀)
- `data/base_feature.csv` — 重新生成 (含全部修复)

### P0: ST标签时间序列化

**问题**: `_build_labels()` 使用 `is_st` 快照，同一ST股票所有季度 target=1，数据泄漏。

**修复**: 多数据源优先级 → Tushare `namechange` (全市场) → akshare `stock_info_sz_change_name` (仅深市) → is_st 快照。

当前 Tushare API 限频 (1次/小时)，使用 akshare SZSE 简称变更数据：
- 7,387 条历史简称变更 → 1,437 条 ST/*ST 记录
- 构建每只股票的 ST 起止时间区间
- `shift(-1)` 生成 "下一季度是否 ST" 标签

**效果**: 正样本 4.83% → 1.29%，200 只股票 target 有时序变化（进入/退出 ST）

**注意**: 仅覆盖深市 (.SZ)，沪市 (.SH) 暂用 is_st 快照。Tushare API 恢复后全市场覆盖。

### P1: 市场数据后缀修复

**问题**: `market_quarterly.csv` 用 `.SS` (yfinance)，`base_feature.csv` 用 `.SH` (Tushare)，导致 2,314 只上海股市场数据 100% 缺失。总覆盖率 49%。

**修复**:
- `market_quarterly.py:64` 添加后缀归一化: `.SS` → `.SH` (与 data_fetcher.py:148 一致)
- 重新运行 `market_quarterly.py` 生成正确后缀的 market_quarterly.csv

**效果**: 市场数据覆盖率 49% → 90.1%。沪市 NaN 率 100% → 5.3%。

### P2: 极端值 Winsorize

**问题**: interest_coverage (11.8% 极端), cf_to_netprofit (3.8% 极端) 等未截尾。

**修复**: `_handle_missing()` 末尾对 44 个比率列做 1%/99% clip。

**效果**: 所有比率列极端值已截尾，min=q01, max=q99。

### P0 补充: baostock 全市场 ST 标签

Tushare `namechange` 和 `bak_basic` 均受 1次/小时 限频，无法在单次会话内完成查询。

**替代方案**: 使用 **baostock** `query_history_k_data_plus` 的 `isST` 字段 + akshare SZSE 简称变更，构建全市场时间序列 ST 标签：
- baostock: 932只沪市股票的历史 ST 状态 (免费, 无频率限制)
- akshare: 453只深市股票的 ST/*ST 历史区间 (1,437条变更记录)
- 其余股票保守标记 is_st=0

**最终 P0 效果**:
- 正样本: 1,846 (1.87%)，278只股票 target 有时序变化
- 季度分布: 1.3%-3.1%，2024Q4-2025Q4 略高 (监管趋严)
- SH覆盖: 2,373只 | SZ覆盖: 2,693只 | ST数据来源: baostock(932) + akshare(453)
