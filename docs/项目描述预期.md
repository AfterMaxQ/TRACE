### 1. 引言

#### 1.1 编写目的
本需求分析文档旨在明确 **TRACE** 平台的功能、性能、数据和技术方案，为项目开发、测试、验收提供统一依据。文档面向项目开发者（本人）、潜在面试官及技术评审人，通过结构化描述确保项目方向准确、边界清晰。

#### 1.2 项目背景
- **行业痛点**：传统企业信用风险评估过于依赖财务报表，存在滞后性；宏观环境、供应链传导、市场舆情等实时信息未被有效融合；风险传导路径不透明，难以预警连锁反应。
- **个人定位**：系统设计者具备计算机科学基础与国际贸易经济学知识，期望将多源数据融合、机器学习、图神经网络、自然语言处理与AI对话能力结合，构建一个既满足学术深度又具备实际应用价值的智能风控平台，作为金融数据分析方向的求职作品。
- **差异化价值**：市场上多数信用风险项目仅使用单一财务模型，而本系统深度融合了 **供应链传导图神经网络** 与 **NLP舆情情感修正**，并引入 **AI Agent** 自然语言交互，特别擅长捕捉外贸企业受汇率、BDI、关税等国际贸易因素冲击的动态风险。

---

### 2. 项目概述

#### 2.1 产品定义
TRACE 是一个基于 Python 的多源数据融合智能风控平台，为分析师、基金经理、风控部门提供企业违约风险评分、供应链传导预警、宏观‑舆情联动分析与自然语言交互报告功能。

#### 2.2 目标用户
- **主要用户**：金融机构信用研究员、银行风控人员、基金投资经理；
- **次要用户**：企业供应链管理部门（查看自身及上下游风险）；
- **演示用户**：面试官、技术评审人。

#### 2.3 核心目标
1. **违约预测**：融合财务报表、市场行情、宏观指标，通过 XGBoost 等模型输出高精度违约概率；
2. **供应链传导建模**：构建企业知识图谱，利用图神经网络（GAT）量化风险传导效应；
3. **动态舆情修正**：引入 NLP 情感分析，对突发事件快速调整风险评分；
4. **国际贸易特色**：集成进出口品类、汇率、BDI、关税冲击模拟，突出对跨国供应链的分析优势；
5. **智能交互**：借助大语言模型 Agent，实现自然语言查询、归因分析、报告自动生成。

#### 3.2 功能详述

**F‑01 数据管理**
- 采集上市公司财报（资产负债表、利润表、现金流量表），频率季度；
- 采集宏观指标（GDP、CPI、PMI、M2、1年/10年期国债收益率、Shibor），频率月度，按季度平均对齐；
- 采集市场行情（日线 OHLCV），计算季度波动率、Beta、最大回撤等；
- 采集舆情文本数据：主流财经新闻标题/摘要、公司公告、券商研报标题；
- 采集供应链关系：从年报、公告中提取供应商‑客户名称，构建映射；
- 贸易数据：海关进出口金额、BDI指数、主要货币汇率、美国/欧盟关税变动；
- 存储采用关系型数据库（SQLite/PostgreSQL），便于快速查询。

**F‑02 传统特征工程**
- 计算 >30 个财务比率（流动比率、速动比率、资产负债率、利息保障倍数、ROE、ROA、毛利率、增长类指标等）；
- 标签定义：下一季度被交易所标记为 ST 或 *ST → 1（高风险），否则 → 0；
- 加入宏观特征：GDP同比、CPI、PMI、M2同比、利率变动；
- 加入市场特征：季度收益率、季度波动率、最大回撤、Beta、价格相对强度等。

**F‑03 NLP 舆情分析**
- 使用中文金融预训练模型（如 FinBERT‑zh）或微调 distilbert-base‑multilingual，对文本进行情感三分类（正面/中性/负面）；
- 按公司‑日期聚合得到每日情感得分（‑1至1），再按季度提取统计量：均值、标准差、负面占比；
- 构建“舆情冲击指数”：若突发负面事件（如被立案调查、债券违约），短期内大幅下调风险修正权重；
- 输出将作为模型的一个分支特征。

**F‑04 供应链与贸易特征**
- 基于年报分行业/分地区收入，计算海外依赖度（海外收入/总收入）；
- 通过 HS 编码匹配主要进出口产品，计算海关出口金额同比波动；
- 构建汇率敏感系数：滚动回归公司股价变动对美元/人民币汇率的弹性；
- 引入波罗的海干散货指数（BDI），计算其与公司营收（或股价）的相关性；
- **关税冲击模拟功能**：用户可在前端输入目标税率变动，系统利用进出口结构估算对利润率的冲击，并映射到综合评分。

**F‑05 知识图谱构建**
- 节点：上市公司、非上市供应商（如有数据）、行业节点；
- 边类型：供应商‑客户、股权质押、同一行业、同一地域；
- 节点属性：财务特征向量、行业编码、规模标签；
- 使用 NetworkX 或 DGL 构建图对象，支持子图查询；
- 图数据定期更新，与财报发布频率同步。

**F‑06 违约预测模型**
- 训练集时间：2018‑2022，测试集：2023；
- 处理类别不平衡（SMOTE过采样 + 样本权重）；
- 尝试 XGBoost、LightGBM，选择交叉验证 AUC 最优；
- 输出违约概率（0‑1）、风险评级（如 AAA～D）。

**F‑07 图神经网络**
- 将上述图结构输入 GAT（Graph Attention Network）；
- 节点初始特征包含财务特征、宏观特征及基础违约概率；
- 通过多层注意力传播，输出每个节点的 **传导风险得分**；
- 模型目标可以设置为有监督：标签为下一季度是否发生风险事件（ST或大幅下跌），或者无监督重建节点特征并利用异常检测；
- 注意力权重用于可视化传导路径强度。

**F‑08 模型融合**
- 综合评分 = 基础违约概率 × 0.6 + 传导风险得分 × 0.3 + 舆情修正系数 × 0.1；
- 各权重可通过验证集校准；
- 最终输出 0‑100 的风险评分，映射到评级（低风险、关注、预警、高风险）。

**F‑09 AI Agent 对话**
- 使用 LangChain 搭建 React Agent，后端接入大模型 API（如通义千问‑Plus 或 GPT‑4o mini）；
- 定义工具函数（Tool）：
  - `query_risk_score(company_code)` → 返回最新综合评分及评级；
  - `trace_supply_chain(company_code)` → 返回上下游公司及传导风险；
  - `analyze_macro(quarter)` → 返回宏观解读要点；
  - `simulate_tariff(company_code, rate_change)` → 运行关税模拟并返回影响；
  - `generate_report(company_code)` → 整合信息，生成 Markdown 格式风险简报并导出 PDF。
- Agent 接受自然语言输入，自动调用相应工具，返回结果与解释。

**F‑10 可视化仪表盘**
- 首页：宏观指标走势图（GDP、PMI、利率等），使用 Plotly 交互图表；
- 第二页：公司风险查询 → 输入代码，展示评分卡、历史趋势、财务比率仪表盘；
- 第三页：PCA 降维散点图（按风险聚集），可悬停查看公司名称；
- 第四页：供应链网络图（D3.js 或 PyVis），节点大小表示风险，边粗细表示传导强度；
- 侧边栏嵌入 AI Agent 对话窗口，以聊天形式交互；
- 技术支持 Dashboard 页面响应速度 < 1 秒（本地查询）。

**F‑11 系统部署**
- 使用 Streamlit Cloud 或云服务器（阿里云/腾讯云免费实例）部署；
- 代码托管在 GitHub，包含完整 README 和 requirements.txt；
- 支持离线数据包运行，或在线调用 API（舆情、大模型）更新。

## 第1天：多源数据采集与初步清洗 (6h)

**上午：获取结构化数据 (3h)**

- **1.1 财务数据下载 (1.5h)**
  - 注册Tushare Token，调用`income`, `balancesheet`, `cashflow`接口
  - 下载2018Q1-2023Q4所有A股上市公司数据，合并为三张表
  - 保存为`financials_raw.csv`或直接写入SQLite表`raw_financials`

- **1.2 行情数据获取与季度特征预计算 (1h)**
  - 使用`akshare`或`tushare`获取日线行情（全部A股）
  - 按季度聚合计算：季度对数收益率、季度内日收益率波动率、最大回撤
  - 计算各股票相对于沪深300指数的季度Beta（使用过去12个月日收益率滚动回归）
  - 存入`market_quarterly`表

- **1.3 宏观指标获取 (0.5h)**
  - 通过`akshare`宏观模块下载：GDP当季同比、CPI当月同比、制造业PMI、M2同比、1年期国债收益率、Shibor隔夜利率
  - 将月度数据重采样为季度均值，生成`macro_quarterly.csv`

**下午：非结构化数据与供应链爬取 (3h)**

- **1.4 舆情文本获取 (1.5h)**
  - 确定数据源：财联社电报、新浪财经个股新闻（可通过`akshare`新闻接口获取历史新闻）
  - 编写爬虫脚本（可利用AI生成）抓取2020年至今的公司新闻标题、日期、股票代码
  - 去重、清洗HTML标签，存入`news_raw`表

- **1.5 供应链关系数据采集 (1h)**

## 第2天：传统特征工程与标签构建 (6h)

**上午：财务特征与标签 (3h)**

- **2.1 财务比率计算 (1.5h)**
  - 编写`src/features.py`，使用Pandas向量化计算40+金融指标：
    - 流动比率、速动比率、现金比率、资产负债率、权益乘数、利息保障倍数
    - 存货周转率、应收账款周转率、总资产周转率
    - ROE、ROA、销售毛利率、营业利润率、净利润率
    - 营业收入同比增长率、净利润同比增长率、总资产增长率
    - 经营活动现金流/营业收入、自由现金流/总负债
  - 将所有比率与原始数据拼接为单条记录（按公司-季度）

- **2.2 定义违约标签 (0.5h)**
  - 从`tushare`获取历史ST/撤ST日期
  - 生成“下一季度是否ST”的二元标签 `target`
  - 分析标签平衡度，确认正样本比例（一般<5%）

- **2.3 宏-微观数据融合 (1h)**
  - 将宏观指标按季度合并到每一行公司数据
  - 将市场特征（波动率、Beta等）也合并进去
  - 生成“基础特征宽表”`base_feature.csv`

**下午：外贸特色因子 (2h) 与特征降维 (1h)**

- **2.4 计算外贸/供应链敏感度指标 (1.5h)**
  - 基于年报分地区收入计算海外收入占比（需手动整理少量样本或利用`tushare`的`fina_indicator`字段）
  - 利用汇率与公司股价数据估计汇率暴露（滚动回归，可提前计算好Beta存入特征）
  - 计算BDI季度变化率
  - 构建“关税冲击模拟函数”雏形：假设关税增加10%，估算成本影响 = 海外收入占比 * 0.1 * (1-毛利率)，映射为预期利润率下降幅度，进而影响Z-score得分
  - 将上述各指标加入特征集

- **2.5 PCA降维与可视化准备 (0.5h)**

  - 生成PCA散点图所需数据（PC1, PC2, 公司名, 标签）

**晚上：特征集定稿 (1h)**
- 处理缺失值（行业均值填充、前向填充）
- 保存最终建模宽表`model_data.parquet`（高效读写）

## 第3天：NLP舆情引擎与知识图谱构建 (6h)

**上午：舆情情感分析 (3h)**

- **3.1 文本预处理与微调样本准备 (1h)**
  - 加载新闻数据，拼接标题+摘要
  - 随机抽取2000条人工标注（可借助AI辅助），分为正面/负面/中性
  - 划分训练集和验证集

- **3.2 微调中文金融情感模型 (1.5h)**
  - 使用`transformers`加载`bert-base-chinese`或`FinBERT-Chinese`（如`luhua/chinese_pretrain_mrc_macbert_large`结合情感分类头）
  - 进行3-5个epoch微调，保存最佳模型
  - 若算力不足，直接使用零样本分类pipeline（`facebook/bart-large-mnli`）进行情感推理，或调用现成的情感分析API（节省时间）

- **3.3 生成动态舆情特征 (0.5h)**
  - 对所有新闻进行情感推理，得到逐条得分
  - 按公司-季度聚合：`mean_sentiment`, `std_sentiment`, `negative_ratio`, `latest_shock_score`（最近30天内负面冲击强度）
  - 保存为`sentiment_features.csv`

**下午：知识图谱构建 (3h)**

- **3.4 构建企业-供应链图谱 (2h)**

  - 边列表：基于`supply_chain_edges.csv`生成“供应商-客户”关系；添加“同一行业”虚拟边（权重低，用于平滑传导）、“股权质押”关系（从数据库抽）
  - 使用`networkx`构建有向图，计算节点中心度：度、PageRank、集聚系数、邻居平均风险（先用基础标签初步计算）
  - 将图导出为DGL格式图（`dgl.heterograph`或`dgl.graph`），节点特征为财务特征+基础风险概率

- **3.5 图特征提取 (1h)**
  - 计算每个节点在图中的统计特征：出度、入度、PageRank、聚类系数
  - 计算基于一阶邻居的平均违约概率（作为简单传导指标）
  - 将这些特征也加入后续的XGBoost特征集中，以备GNN效果不好时使用

## 第4天：基础违约模型训练 (XGBoost) (6h)

- **4.1 数据集划分与预处理 (1h)**

## 第5天：图神经网络风险传导模型 (6h)

- **5.1 准备图数据输入 (1.5h)**
  - 将步骤3.4构建的图与节点特征合并，确保节点特征维度一致
  - 划分训练/测试节点（注意图结构不切断，只分割节点标签）
  - 将图保存为DGL的`heterograph`二进制文件

- **5.2 搭建GAT模型 (2h)**
  - 使用DGL实现2层GAT（Graph Attention Network），隐藏层维度64，注意力头数4
  - 输出层：二分类（违约/正常）
  - 损失函数：加权交叉熵（因不平衡）
  - 定义训练循环，早停策略

- **5.3 训练与性能评估 (2h)**
  - 训练100-200 epoch，监控验证集AUC
  - 测试集评估：AUC、准确率
  - 可视化注意力权重：选取几个高风险节点，查看其邻居的注意力分布，解释传导路径
  - 保存最佳模型`models/gat_model.pth`

- **5.4 提取传导风险得分 (0.5h)**
  - 对全部节点推理，输出`contagion_prob`（风险传导概率）
  - 将得分合并到原始数据集

## 第6天：模型融合与综合评分卡 (6h)

**上午：融合策略设计 (3h)**

- **6.1 生成舆情修正因子 (1h)**
  - 将第3天得到的`latest_shock_score`归一化，并转化为修正系数：`factor = 1 + 0.1 * shock_score`（范围约0.9~1.1）
  - 对基础概率进行初步修正：`adj_base_prob = base_prob * factor`

- **6.2 训练Stacking集成模型 (2h)**
  - 以`adj_base_prob`, `contagion_prob`, 以及部分原始强特征（如资产负债率、ROE）作为元特征
  - 使用逻辑回归作为元模型，训练融合权重（或直接用线性模型确定系数）
  - 交叉验证评估融合后AUC是否提升

**下午：构建最终评分卡 (2h) & 模拟测试 (1h)**

- **6.3 评分刻度映射 (1.5h)**
  - 将融合后的违约概率映射为风险评分：`Score = 100 - 100*prob`（调整密度使分布均匀）
  - 定义风险等级：低风险(80-100)、关注(60-80)、预警(40-60)、高风险(0-40)
  - 为每个公司-季度记录生成评分与等级，保存`final_score.csv`

- **6.4 关税冲击模拟功能完善 (0.5h)**
  - 在`src/utils.py`中实现函数`simate_tariff_impact(company, tariff_increase)`：
    - 获取公司海外收入占比、毛利率
    - 计算利润率减少量 = 海外收入占比 * 关税增幅 * (1 - 毛利率)
    - 重新估算财务指标，预测新违约概率
    - 返回风险评分变化值

- **6.5 系统测试与准确性验证 (1h)**
  - 抽查10-20家公司，检查评分是否符合直觉（如高风险公司财务困难）
  - 记录预定义测试案例，确保模型逻辑正确

## 关键注意事项与补充建议
1. **数据合规性**：所有数据源必须合法公开，爬虫设置合理频率，避免侵权。
2. **供应链数据真实性**：若供应商-客户数据难以大量获取，应明确声明关系为“基于公开信息与合理推断的示例”，并在演示时重点展示5-10家企业的完整传导案例，辅以其他企业的统计模拟。
3. **模型可解释性**：在Dashboard中加入SHAP解释图或LIME，增强业务人员信任。
4. **Agent功能节制**：Agent输出应基于实际数据，避免生成虚假报告，必要时加入“回答依据”标注。
5. **性能**：Streamlit应用可以考虑使用`session_state`保持连接，减少重复加载。
6. **备份与版本控制**：每日结束前commit并push，使用分支开发新功能。

这个详细步骤清单涵盖了从数据到部署的全链路，并针对你的背景和求职目标进行了深度定制，确保项目既有技术高度又有业务落地价值。严格按照步骤执行，两周内可以完成一个令人印象深刻的作品。
