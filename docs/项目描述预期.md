### 1. 引言

#### 1.1 编写目的
本需求分析文档旨在明确 **TRACE** 平台的功能、性能、数据和技术方案，为项目开发、测试、验收提供统一依据。文档面向项目开发者（本人）、潜在面试官及技术评审人，通过结构化描述确保项目方向准确、边界清晰。

#### 1.2 项目背景
- **行业痛点**：传统企业信用风险评估过于依赖财务报表，存在滞后性；宏观环境、供应链传导、市场舆情等实时信息未被有效融合；风险传导路径不透明，难以预警连锁反应。
- **个人定位**：系统设计者具备计算机科学基础与国际贸易经济学知识，期望将多源数据融合、机器学习、图神经网络、自然语言处理与AI对话能力结合，构建一个既满足学术深度又具备实际应用价值的智能风控平台，作为金融数据分析方向的求职作品。
- **差异化价值**：市场上多数信用风险项目仅使用单一财务模型，而本系统深度融合了 **供应链传导图神经网络** 与 **NLP舆情情感修正**，并引入 **AI Agent** 自然语言交互，特别擅长捕捉外贸企业受汇率、BDI、关税等国际贸易因素冲击的动态风险。

---

### 2. 项目概述

#### 2.1 产品定义
TRACE 是一个基于 Python 的多源数据融合智能风控平台，为分析师、基金经理、风控部门提供企业违约风险评分、供应链传导预警、宏观‑舆情联动分析与自然语言交互报告功能。

#### 2.2 目标用户
- **主要用户**：金融机构信用研究员、银行风控人员、基金投资经理；
- **次要用户**：企业供应链管理部门（查看自身及上下游风险）；
- **演示用户**：面试官、技术评审人。

#### 2.3 核心目标
1. **违约预测**：融合财务报表、市场行情、宏观指标，通过 XGBoost 等模型输出高精度违约概率；
2. **供应链传导建模**：构建企业知识图谱，利用图神经网络（GAT）量化风险传导效应；
3. **动态舆情修正**：引入 NLP 情感分析，对突发事件快速调整风险评分；
4. **国际贸易特色**：集成进出口品类、汇率、BDI、关税冲击模拟，突出对跨国供应链的分析优势；
5. **智能交互**：借助大语言模型 Agent，实现自然语言查询、归因分析、报告自动生成。

#### 3.2 功能详述

**F‑01 数据管理**
- 采集上市公司财报（资产负债表、利润表、现金流量表），频率季度；
- 采集宏观指标（GDP、CPI、PMI、M2、1年/10年期国债收益率、Shibor），频率月度，按季度平均对齐；
- 采集市场行情（日线 OHLCV），计算季度波动率、Beta、最大回撤等；
- 采集舆情文本数据：主流财经新闻标题/摘要、公司公告、券商研报标题；
- 采集供应链关系：从年报、公告中提取供应商‑客户名称，构建映射；
- 贸易数据：海关进出口金额、BDI指数、主要货币汇率、美国/欧盟关税变动；
- 存储采用关系型数据库（SQLite/PostgreSQL），便于快速查询。

**F‑02 传统特征工程**
- 计算 >30 个财务比率（流动比率、速动比率、资产负债率、利息保障倍数、ROE、ROA、毛利率、增长类指标等）；
- 标签定义：下一季度被交易所标记为 ST 或 *ST → 1（高风险），否则 → 0；
- 加入宏观特征：GDP同比、CPI、PMI、M2同比、利率变动；
- 加入市场特征：季度收益率、季度波动率、最大回撤、Beta、价格相对强度等。

**F‑03 NLP 舆情分析**
- 使用中文金融预训练模型（如 FinBERT‑zh）或微调 distilbert-base‑multilingual，对文本进行情感三分类（正面/中性/负面）；
- 按公司‑日期聚合得到每日情感得分（‑1至1），再按季度提取统计量：均值、标准差、负面占比；
- 构建“舆情冲击指数”：若突发负面事件（如被立案调查、债券违约），短期内大幅下调风险修正权重；
- 输出将作为模型的一个分支特征。

**F‑04 供应链与贸易特征**
- 基于年报分行业/分地区收入，计算海外依赖度（海外收入/总收入）；
- 通过 HS 编码匹配主要进出口产品，计算海关出口金额同比波动；
- 构建汇率敏感系数：滚动回归公司股价变动对美元/人民币汇率的弹性；
- 引入波罗的海干散货指数（BDI），计算其与公司营收（或股价）的相关性；
- **关税冲击模拟功能**：用户可在前端输入目标税率变动，系统利用进出口结构估算对利润率的冲击，并映射到综合评分。

**F‑05 知识图谱构建**
- 节点：上市公司、非上市供应商（如有数据）、行业节点；
- 边类型：供应商‑客户、股权质押、同一行业、同一地域；
- 节点属性：财务特征向量、行业编码、规模标签；
- 使用 NetworkX 或 DGL 构建图对象，支持子图查询；
- 图数据定期更新，与财报发布频率同步。

**F‑06 违约预测模型**
- 训练集时间：2018‑2022，测试集：2023；
- 处理类别不平衡（SMOTE过采样 + 样本权重）；
- 尝试 XGBoost、LightGBM，选择交叉验证 AUC 最优；
- 输出违约概率（0‑1）、风险评级（如 AAA～D）。

**F‑07 图神经网络**
- 将上述图结构输入 GAT（Graph Attention Network）；
- 节点初始特征包含财务特征、宏观特征及基础违约概率；
- 通过多层注意力传播，输出每个节点的 **传导风险得分**；
- 模型目标可以设置为有监督：标签为下一季度是否发生风险事件（ST或大幅下跌），或者无监督重建节点特征并利用异常检测；
- 注意力权重用于可视化传导路径强度。

**F‑08 模型融合**
- 综合评分 = 基础违约概率 × 0.6 + 传导风险得分 × 0.3 + 舆情修正系数 × 0.1；
- 各权重可通过验证集校准；
- 最终输出 0‑100 的风险评分，映射到评级（低风险、关注、预警、高风险）。

**F‑09 AI Agent 对话**
- 使用 LangChain 搭建 React Agent，后端接入大模型 API（如通义千问‑Plus 或 GPT‑4o mini）；
- 定义工具函数（Tool）：
  - `query_risk_score(company_code)` → 返回最新综合评分及评级；
  - `trace_supply_chain(company_code)` → 返回上下游公司及传导风险；
  - `analyze_macro(quarter)` → 返回宏观解读要点；
  - `simulate_tariff(company_code, rate_change)` → 运行关税模拟并返回影响；
  - `generate_report(company_code)` → 整合信息，生成 Markdown 格式风险简报并导出 PDF。
- Agent 接受自然语言输入，自动调用相应工具，返回结果与解释。

**F‑10 可视化仪表盘**
- 首页：宏观指标走势图（GDP、PMI、利率等），使用 Plotly 交互图表；
- 第二页：公司风险查询 → 输入代码，展示评分卡、历史趋势、财务比率仪表盘；
- 第三页：PCA 降维散点图（按风险聚集），可悬停查看公司名称；
- 第四页：供应链网络图（D3.js 或 PyVis），节点大小表示风险，边粗细表示传导强度；
- 侧边栏嵌入 AI Agent 对话窗口，以聊天形式交互；
- 技术支持 Dashboard 页面响应速度 < 1 秒（本地查询）。

**F‑11 系统部署**
- 使用 Streamlit Cloud 或云服务器（阿里云/腾讯云免费实例）部署；
- 代码托管在 GitHub，包含完整 README 和 requirements.txt；
- 支持离线数据包运行，或在线调用 API（舆情、大模型）更新。

## 第1天：多源数据采集与入库

### 步骤5：获取上市公司财务数据 (Tushare/AKShare)
- 获取全A股2018-2023年资产负债表、利润表、现金流量表。
- 统一字段名，写入 SQLite 表 `financials` (code, report_date, 各项指标)。

### 步骤6：下载日线行情并计算季度衍生指标
- 获取前复权日线，按季度计算：
  - 季度收益率 = (季度末收盘价/上季度末收盘价 - 1)
  - 季度波动率 = 日对数收益率标准差 * sqrt(交易日数)
  - 最大回撤
  - Beta (对沪深300指数，使用日收益率回归)
- 入库 `market_quarterly`。

### 步骤7：采集宏观指标
- 来源：国家统计局、东方财富等公开接口。
- 指标：GDP同比增长、CPI、PMI、M2同比增速、1年期国债收益率、10年期国债收益率、美元兑人民币汇率。
- 均为月度数据，取季度平均值。写入 `macro_quarterly`。

### 步骤8：爬取新闻舆情数据 (合规优先)
- 使用开源财经新闻数据集 (如 THUCNews 金融子集) 或 AI 辅助编写爬虫抓取新闻标题 + 摘要 + 时间。
- 每家公司至少覆盖50条历史新闻，存储 `news_raw` 表 (code, date, title, content, source)。

### 步骤9：整理供应链关系
- 从年报、券商研报、天眼查等公开渠道提取前5大客户/供应商关系。
- 手动构建 CSV (最少300对关系)，列：`company_code, related_company_code, relation_type (supplier/customer), weight`。
- 导入 `supply_chain` 表。

### 步骤10：采集国际贸易特色数据
- 获取波罗的海干散货指数 BDI (日, 取季度均值)。
- 记录美国对中国部分商品关税税率历史变化（可选简化，采用新闻报道关键节点）。
- 中欧贸易相关指数（如集装箱运价指数）。
- 存入 `trade_indicators`。

### 步骤11：数据清洗与质量报告
- 剔除ST或*ST超过2年的公司。
- 填补缺失值：行业均值或前向填充；记录缺失率。
- 所有表建立索引 (code, report_date)。

## 第2天：特征工程 (含贸易特征)

### 步骤12：计算财务比率 (30+)
- 流动性：流动比率、速动比率、现金比率。
- 杠杆：资产负债率、权益乘数、利息保障倍数。
- 营运能力：存货周转率、应收账款周转率、总资产周转率。
- 盈利能力：ROE (杜邦分解)、ROA、毛利率、净利率。
- 增长率：营收同比增长率、净利润同比增长率、总资产增长率。
- 现金流：经营现金流/营业收入、自由现金流/负债。

### 步骤13：构建违约标签
- 标签定义：未来两个季度内被 ST 或 *ST → 1 (高风险)，否则 0。
- 标签需领先一期，避免使用当前季度状态。

### 步骤14：合并宏观与市场特征
- 按 code, quarter 关联财务、宏观、市场特征，形成基础宽表 (约60维)。
- 添加行业分类 (申万一级行业) 作为 category 特征编码。

### 步骤15：外贸依赖度特征
- 根据年报“主营业务分地区”数据计算海外收入占比。
- 汇率敏感系数：用过去12个季度的股价收益率对美元兑人民币汇率变动做回归，得到弹性系数。
- BDI波动率：BDI季度变化率的滚动标准差。
- 关税冲击历史值 (可选)：若有关税大幅上调事件，对应季度标记为1。

### 步骤16：特征缩放与存储
- 将所有数值特征标准化/归一化 (训练集统计量)。
- 保存特征列名列表、均值标准差至 `models/scaler.pkl`。

## 第3天：NLP 舆情特征生成

### 步骤17：文本预处理
- 生成样本标注：抽取1000条新闻，人工标注积极/消极/中性 (可借助已有情感词典辅助)。
- 切分训练验证集。

### 步骤18：微调情感分类模型
- 使用 `ProsusAI/finbert` (若英文或可适应金融文本) 或 `bert-base-chinese` 微调。
- 训练3-5个epoch，保存模型 `models/sentiment_model`。
- 推理所有新闻，得到每条的正面/负面概率，计算情感得分 = P(pos) - P(neg)。

### 步骤19：季度舆情特征聚合
- 按公司、季度聚合：
  - 平均情感得分 (mean_sentiment)
  - 情感得分标准差 (std_sentiment)
  - 负面新闻比例 (neg_ratio)
  - 最近一个月负面冲击强度 (冲击期负面概率均值)
- 存入表 `sentiment_quarterly`。

### 步骤20：构建舆情冲击修正系数
- 定义规则：若`neg_ratio > 0.6` 且 `std_sentiment > 阈值`，冲击系数 = -0.1，否则 0。
- 该系数将叠加到最终评分调整中。

## 第4天：知识图谱与 GNN 训练

### 步骤21：构建图数据对象
- 使用 NetworkX 预构建图，节点属性包含：财务特征向量、行业、规模。
- 边属性：关系类型、权重。
- 转换为 DGL 图，分割训练/测试节点（按时间顺序）。

### 步骤22：生成节点初始特征
- 将上一步基础特征 (标准化后) 作为节点特征矩阵 (n_nodes × d)。
- 可加入图结构特征：度、PageRank值、聚集系数。

### 步骤23：定义 GAT 模型
- 2层 GAT，每层多头注意力 (num_heads=4)。
- 输出维度 = 2 (违约/正常)。
- 损失函数：Focal Loss (处理不平衡) 或加权 CrossEntropy。

### 步骤24：训练与评估
- 按照时间顺序划分训练/验证/测试集 (如：2018-2020训练，2021验证，2022-2023测试)。
- 早停策略。
- 记录测试集 AUC, F1, 传导路径注意力权重。

### 步骤25：保存模型与注意力权重
- 保存 `models/gnn_model.pt`。
- 导出高风险注意力边，用于可视化。

## 第6天：模型融合与综合风险评分

### 步骤31：获取各子模型输出
- 基础违约概率 (XGBoost)：`base_score`
- GNN 传导风险概率：对没有在训练集中的节点，使用最后一次迭代的嵌入做预测，得到 `gnn_score`
- 舆情冲击系数：映射到 [ -0.05, +0.05 ] 区间，记为 `sent_adj`

### 步骤32：加权融合 (或训练元模型)
- 简单加权：`final_prob = w1*base_score + w2*gnn_score + w3*sent_adj`
- 权重可通过网格搜索在验证集上最大化 AUC 确定 (例如 w1=0.6, w2=0.3, w3=0.1)。
- 或使用逻辑回归元模型：输入三个概率，输出最终概率。

### 步骤33：构建0-100评分卡
- 评分映射公式：`Score = 100 - 100 * P_default` (再校准到合理分布)。
- 风险等级：低风险 (>80)、关注 (60-80)、预警 (40-60)、高风险 (<40)。

### 步骤34：保存最终模型与参数
- 导出融合权重、评分映射表、全流程pipeline (sklearn Pipeline 或自定义类)。
- 编写 `run_risk_assessment(code)` 函数，一键输出最终评分、各子项得分。

## 第7天：AI Agent 开发

### 步骤35：工具函数定义 (agent_tools.py)
- `get_risk_score(company_code)`：调用融合模型，返回评分、等级、关键指标。
- `get_supply_chain_risk(company_code)`：查询该公司的上下游，返回其自身及关联方风险简要。
